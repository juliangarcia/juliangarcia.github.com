Software

The core of my code is open-source and available on my GitHub repo. Pull requests are much appreciated. I keep four main open-source projects to study evolutionary dynamics and game theory:

  • PyEvoDyn: Provides a set of simple but general python routines for numerical, symbolic and simulation work (in finite and infinite populations). Check it out in action here, replicating Imhof et al. (2005); and here, replicating Hauert et al (2007).

  • Agent-Based-Evolutionary-Dynamics: Implements an easy-to-extend Wright-Fisher process for individual-based simulations e.g., the model in this paper. It uses Java and is heavily object-oriented.

  • JEvoDyn: Provides out-of-the-box simulations for a Moran Process with any given game. It uses Java and is heavily object-oriented.

  • game_theory.py: a very alpha set of python routines implementing classic solution concepts in game theory.

Some of the projects I have worked on have executables available at evolutionandgames.com.

I use mostly Java and Python (Numpy, IPython, MatPlotLib and Pandas). I use Mathematica for mathematics, but I am trying to fully switch to SymPy. Lately I have been playing with Julia, and it looks very promising. I use Eclipse for Java and Sublime Text for everything else.

I am an advocate of high standards in scientific computing and I try to follow the best practices. Check the excellent tutorials and resources at Software Carpentry if you want to raise the standards of scientific software.
